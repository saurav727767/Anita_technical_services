OTP Security Architecture

Secure OTP Login Sequence

This flow details how OTP-based authentication works securely in our system, preventing brute-force and credential stealing:

Step 1: User Request

User submits email/phone. The backend checks if the account is rate-limited. If clean, it generates a cryptographically secure 6-digit code.

Step 2: Delivery & Storage

Code is hashed (e.g., using SHA256 or bcrypt) and saved in temporary database with an expiry timestamp (typically 2-5 minutes). The raw code is dispatched to user via SMS API (like Twilio) or Email API (like SendGrid).

Step 3: Verification & Lockout

User enters code. Backend compares hash. If correct, user receives a Session JWT. If incorrect, attempt count increments. After 3 failures, login is blocked for 30s-10m to block automated bots.

Security Tip

Never return the generated OTP code in the frontend response body. OTPs should only exist inside the delivery channel (SMS/Email) and as a hash inside the backend cache database.

OTP Table Schema (SQL Example)

For high performance and clean validation, OTP verification should utilize a separate table or a memory cache (like Redis) with automatic record TTL (Time-To-Live).

CREATE TABLE otp_verifications (
  id VARCHAR(36) PRIMARY KEY,
  client_id VARCHAR(50) NOT NULL,
  otp_hash VARCHAR(255) NOT NULL, -- Stored securely hashed
  attempts INT DEFAULT 0,         -- Counter for rate limiting
  expires_at TIMESTAMP NOT NULL,  -- Verification Window Expiry
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

For production cache delivery (e.g., Redis):

// Store OTP hash with 120 seconds expiry time
await redis.set(`otp:${clientId}`, otpHash, 'EX', 120);

Backend verification controller (NodeJS / Express)

Here is an exemplary Express JS controller showing secure hashing, rate-limiting, and verification validation:

const crypto = require('crypto');
const bcrypt = require('bcrypt');

// Generate and Save OTP
app.post('/api/auth/request-otp', async (req, res) => {
  const { contact } = req.body;
  
  // Rate-limiting check...
  const recentAttempts = await getAttemptsCount(contact);
  if (recentAttempts >= 3) {
    return res.status(429).json({ error: "Lockout active. Try later." });
  }

  // Cryptographically secure 6-digit code
  const code = Math.floor(100000 + crypto.randomInt(900000)).toString();
  const otpHash = await bcrypt.hash(code, 10);
  
  await saveOTPSession({
    contact,
    hash: otpHash,
    expiresAt: Date.now() + 2 * 60 * 1000 // 2 minutes
  });

  // Dispatched code to SMS API (Simulated in response for development only)
  await smsGateway.send(contact, `Your OTP is: ${code}`);
  return res.json({ message: "OTP Sent Successfully." });
});
